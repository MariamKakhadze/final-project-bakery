# BULE — საცხობის ვებსაიტი

ქართულენოვანი საცხობის ვებსაიტი, სადაც მომხმარებელს შეუძლია პროდუქციის ნახვა, კალათაში დამატება და ონლაინ შეკვეთა. საიტი აგებულია როგორც მრავალგვერდიანი სტატიკური ფრონტენდი **Supabase**-ის საფუძველზე — ავტორიზაციისა და პროდუქტების მონაცემებისთვის.

განვითებისას საიტი მუშაობს ადგილობრივად: `http://localhost:3000`

---

## რას აკეთებს პროექტი

### მაღაზია

- **მთავარი** — hero სექცია, „ჩვენს შესახებ“-ის მოკლე ბლოკი და **პოპულარული პროდუქტების** კარუსელი (6 პროდუქტი: თითოეული კატეგორიიდან ერთ-ერთი შემთხვევითი + 2 დამატებითი), ისრებით გადაადგილებით.
- **მენიუ** — სრული კატალოგი Supabase-დან, კატეგორიების ფილტრებით (ფუნთუშები, ტორტები, ნამცხვრები, ცომეული), მობილური ფილტრის პანელით და ძიებით.
- **ჩვენს შესახებ და კონტაქტი** — ბრენდის ისტორია და კონტაქტის ფორმა.

### პროდუქტები

- პროდუქტის ბარათზე ჩანს სურათი, სახელი, ფასი და მოკლე აღწერა.
- ბარათზე დაკლიკებით იხსნება **პროდუქტის მოდალი**, სადაც არის:
  - სრული სურათი, სახელი, ფასი და აღწერა
  - **შემადგენლობა**
  - **კვებითი ღირებულება** (100 გ-ზე)
  - რაოდენობის კონტროლი და **იყიდე** ღილაკი
- თუ პროდუქტს არ აქვს ცალკეული მონაცემები, გამოიყენება კატეგორიის ნაგულისხმევი მნიშვნელობები; კონკრეტული პროდუქტების გადაფარვა შესაძლებელია `js/products.js`-ში.

### კალათა და გადახდა

- პროდუქტის დამატება ბარათიდან ან მოდალიდან.
- კალათის გვერდითი პანელი (drawer) — რაოდენობა, ქვეჯამი, მიტანა და საერთო თანხა.
- კალათა ინახება **sessionStorage**-ში ბრაუზერის სესიის განმავლობაში.
- **გადახდის** გვერდი შეკვეთის დასასრულებლად.

### ავტორიზაცია

- **რეგისტრაცია** — სახელი, დაბადების თარიღი (dd/mm/yyyy კალენდარი), ელ. ფოსტა და პაროლი.
- **შესვლა** — ელ. ფოსტა/პაროლი ან Google OAuth.
- ელ. ფოსტის დადასტურება Supabase-ით (PKCE); დადასტურების შემდეგ მომხმარებელი ბრუნდება აპში.
- შესული მომხმარებელი ხედავს **სახელს პროფილის ხატულას გვერდით**; დაკლიკებით იხსნება მენიუ **გასვლით**.

## როგორ არის აგებული

### არქიტექტურა

```
ბრაუზერი (HTML გვერდები)
    │
    ├── სტატიკური ასეტები (სურათები, ფონტები, აიკონები)
    ├── კომპილირებული CSS (css/main.css)
    └── Vanilla JavaScript მოდულები
            │
            └── Supabase (Auth + PostgreSQL `products` ცხრილი)
```

პროექტში არ გამოიყენება React ან bundler. თითოეული გვერდი — ცალკე HTML ფაილი, რომელიც იტვირთავს საერთო სკრიპტებსა და სტილებს. ლოგიკა დაყოფილია პატარა JS ფაილებად, რომლებიც ერთდება გლობალურ ობიექტებზე (მაგ. `window.BuleProducts`, `window.BuleCart`).

### ტექნოლოგიები

| ფენა | ტექნოლოგია |
|------|------------|
| Markup | HTML5 (`lang="ka"`) |
| სტილები | SCSS → `css/main.css`, ფონტები (BPG Banner Caps, ALK Katerina) |
| ლოგიკა | Vanilla JavaScript (IIFE სტილი) |
| ბექენდი | Supabase (Auth, ბაზა, სურვილისამებრ Edge Functions) |
| ლოკალური გაშვება | [`serve`](https://www.npmjs.com/package/serve), პორტი **3000** |

### გვერდები

| ფაილი | დანიშნულება |
|-------|-------------|
| `index.html` | მთავარი |
| `menu.html` | მენიუ |
| `about.html` | ჩვენს შესახებ |
| `contact.html` | კონტაქტი |
| `sign-in.html` / `sign-up.html` | ავტორიზაცია |
| `payment.html` | გადახდა |

### JavaScript მოდულები

| ფაილი | როლი |
|-------|------|
| `js/supabase-client.js` | Supabase კლიენტი, auth redirect, ელ. ფოსტის დადასტურების ბანერი |
| `js/products.js` | პროდუქტების ჩატვირთვა, შემადგენლობა/კვება, ბარათების შექმნა |
| `js/home.js` | პოპულარული პროდუქტების კარუსელი მთავარ გვერდზე |
| `js/menu.js` | ფილტრები, ძიება, კატეგორიების გადართვა |
| `js/product-modal.js` | პროდუქტის მოდალი (`selectedProduct`, overlay, დახურვა X/Escape-ით) |
| `js/cart.js` | კალათის drawer, sessionStorage, დამატება/განახლება |
| `js/auth.js` | შესვლა, რეგისტრაცია, თარიღის picker, პაროლის ჩვენება |
| `js/auth-ui.js` | სახელი header-ში და გასვლის dropdown |
| `js/nav.js` | მობილური ბურგერ-მენიუ |
| `js/payment.js` | გადახდის გვერდის ლოგიკა |

### სტილები (SCSS)

წყარო ფაილები — `scss/` ფოლდერში, კომპილირდება `css/main.css`-ში:

- `abstracts/` — ცვლადები, mixin-ები
- `base/` — reset, ტიპოგრაფია
- `components/` — navbar, footer, ღილაკები, ბარათები, ფორმები, კალათა, მოდალი
- `pages/` — გვერდის სპეციფიკური სტილები

### ასეტები

- `assets/images/` — პროდუქტისა და ბრენდის სურათები
- `assets/icons/` — UI აიკონები (კალათა, პროფილი, კალენდარი, სოც. ქსელები)
- `assets/fonts/` — ვებ-ფონტები

### Supabase

- **Authentication** — ელ. ფოსტა/პაროლი და Google; PKCE redirect → `http://localhost:3000/`
- **Database** — `products` ცხრილი: `id`, `name`, `price`, `category`, `image`
- **სურვილისამებრ** — `supabase/functions/` და `server/` catalog/orders/health handler-ებისთვის

პროდუქტის დეტალები (შემადგენლობა, კვება) ემატება კლიენტ-ზე `js/products.js`-ში. მომავალში Supabase-ში შეგიძლიათ დაამატოთ `ingredients` და `nutrition` სვეტები და პირდაპირ ბაზიდან ჩატვირთვა.

## გაშვება

### რა გჭირდებათ

- [Node.js](https://nodejs.org/)
- [Supabase](https://supabase.com/) პროექტი

### 1. დამოკიდებულებების დაყენება

```bash
npm install
```

### 2. Supabase-ის კონფიგურაცია

1. შექმენით პროექტი [supabase.com/dashboard](https://supabase.com/dashboard)-ზე.
2. დაამატეთ `products` ცხრილი მინიმუმ ასეთი ველებით: `id`, `name`, `price`, `category`, `image`.
3. პროექტის URL და publishable key ჩაწერეთ `js/supabase-client.js`-ში.
4. **Authentication → URL Configuration**-ში დააყენეთ:
   - **Site URL:** `http://localhost:3000`
   - **Redirect URLs:** `http://localhost:3000/**`

ელ. ფოსტის დადასტურებისა და ლოკალური განვითების შენიშვნები — `.env.example`-ში.

### 3. ლოკალურად გაშვება

```bash
npm run dev
```

გახსენით [http://localhost:3000](http://localhost:3000)

### 4. SCSS-ის კომპილაცია (სურვილისამებრ)

თუ `scss/` ფაილებს შეცვლით, განაახლეთ `css/main.css`:

```bash
npx sass scss/main.scss css/main.css
```

## ძირითადი ფუნქციები

- ადაპტიური (responsive) განლაგება და მობილური ნავიგაცია
- პოპულარული პროდუქტების კარუსელი (დესკტოპზე 3 ხილული, დანარჩენი ისრებით)
- პროდუქტის მოდალი შემადგენლობით და კვებითი ღირებულებით
- სესიაზე დაფუძნებული კალათა
- Supabase ავტორიზაცია — პროფილი header-ში და გასვლა
- რეგისტრაციაზე დაბადების თარიღის არჩევა კალენდრით (dd/mm/yyyy)
- სათადაგომო პროდუქტების კატალოგი, თუ Supabase მიუწვდომელია
